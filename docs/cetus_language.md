# Спецификация языка программирования CETUS

## 1. Основная идея
**CETUS** — это системный компилируемый язык программирования, сочетающий в себе строгую типобезопасность и минимализм Go с современным синтаксическим сахаром популярных скриптовых языков (JS/Python).

**Философия:** «Простота синтаксиса — быстрота написания». CETUS призван минимизировать когнитивную нагрузку на разработчика, предоставляя эффективные инструменты для работы с данными (слайсы, структуры) и лаконичный синтаксис для обработки ошибок и пустых значений.

---

## 2. Базовая лексика и семантика
*   **Типизация:** Статическая. Основные типы: `int` (64-бит), `string` (immutable), `bool`.
*   **Управление памятью:** Ссылочные типы (слайсы `[]Type` и структуры `struct`). Наличие значения `nil`.
*   **Структура программы:** Обязательное наличие пакета `main` и функции `main()`.
*   **Области видимости:** Уровень пакета, функции и блока `{}`.
*   **Ограничения:** До 256 локальных переменных в функции, до 1024 полей в структуре.

## 3. Классификация новых фич по сложности

### Низкая сложность (1–2 дня)
*   **Null-coalescing (`??`):** Краткая проверка на `nil`.
*   **Wildcard в match (`_`):** Обработка случая «по умолчанию».

### Средняя сложность (3–6 дней)
*   **Опциональная цепочка (`?.`):** Безопасный доступ к полям структур.
*   **Интерполяция строк (`{var}`):** Удобное формирование строк.
*   **Множественные возвраты:** Возврат нескольких значений из функции (как в Go).
*   **Структуры (`struct {}`):** Базовая реализация пользовательских типов.
*   **defer:** Отложенный вызов для освобождения ресурсов.
*   **Охранники (guards) в match (`if`):** Дополнительные условия в ветках выбора.

### Высокая сложность (7–20+ дней)
*   **Pattern matching (базовый):** Конструкция `match/case`.
*   **Деструктуризация в match:** Извлечение значений из структур/слайсов прямо в `case`.
*   **Модульная система (`import/export`):** Управление зависимостями между файлами.
*   **Конкурентность (горутины):** Многопоточное выполнение (самая рискованная фича).

## 4. План реализации (Roadmap)

### План «МИНИМУМ» (База для курсовой)
*Фокус на работоспособности ядра и минимальном удобстве.*
1.  **Ядро:** Переменные (`var`, `:=`), циклы (`for range`), условия (`if/else`).
2.  **Типы:** Целые числа, строки, слайсы и базовые функции (`len`, `append`).
3.  **Улучшения:**
    *   Null-coalescing (`??`) — крайне полезно при наличии `nil`.
    *   Сырые строки (`r"..."`) — просто реализовать в лексере.
    *   Множественные возвраты из функций (фундамент для обработки ошибок).

### План «НОРМАЛ» (Полноценный язык)
*Добавление структурной организации и современного синтаксиса.*
1.  **Структуры (`struct`):** Доступ к полям через точку.
2.  **Ресурсный менеджмент:** Оператор `defer`.
3.  **Modern DX (Developer Experience):**
    *   Интерполяция строк.
    *   Опциональная цепочка (`?.`) — предотвращает падения при работе со структурами.
4.  **Pattern matching (Light):** Простой `match` по значениям с использованием `_` (wildcard).

### План «МАКСИМУМ» (Флагманский уровень)
*Язык, претендующий на реальное использование.*
1.  **Advanced Pattern Matching:** Полная реализация с деструктуризацией и guard-условиями.
2.  **Модульность:** Реализация `import` для разделения кода на файлы.
3.  **Синтаксический сахар:** Полная поддержка всех операторов из таблицы.
4.  **Конкурентность:** (Опционально, если закрыты все остальные пункты) Реализация `go routine`.

## 5. Резюме синтаксиса (Пример кода)

```go
package main

struct User {
    name string
    age  int
}

func fetchUser(id int) (User, bool) {
    // ... логика
    return User{name: "Alice", age: 25}, true
}

func main() {
    user, ok := fetchUser(1)
    
    // Интерполяция и Null-coalescing
    print("User: {user?.name ?? "Guest"}") 

    // Pattern matching с деструктуризацией
    match user {
        case User{age: a} if a < 18 => print("Minor")
        case User{name: "Admin"}    => print("Root")
        case _                      => print("Default user")
    }

    defer print("Done")
}
```

Этот документ объединяет академические требования твоей презентации и практические цели из таблицы фич, создавая четкий вектор развития проекта.
