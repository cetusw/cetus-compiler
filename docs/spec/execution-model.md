## Фазы выполнения

Текущий frontend-пайплайн:

1. чтение исходного текста;
2. лексический анализ;
3. синтаксический анализ;
4. построение AST через semantic actions;
5. семантический анализ.

## Целевая TDD-модель

Целевая модель выполнения:

1. собрать программу;
2. найти тесты;
3. выполнить тесты в изоляции;
4. проверить assertions, properties и coverage;
5. при успехе передать управление entry point.

## Условия запуска программы

Программа не должна запускаться, если:

- есть лексические ошибки;
- есть синтаксические ошибки;
- есть семантические ошибки;
- не прошёл хотя бы один тест;
- не выполнены обязательные правила покрытия.
