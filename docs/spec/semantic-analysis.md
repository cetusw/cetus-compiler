## Назначение

Семантический анализ проверяет программу после построения AST.

## Аннотирование AST

Каждый `ASTNode` может хранить выведенный тип:

```cpp
std::optional<Type>
```

Семантический анализатор записывает типы в существующие AST-узлы через `SetInferredType`.

## Таблица символов

`SymbolTable` хранит соответствие имени и семантического символа. Символ содержит имя и тип.

Идентификатор должен быть определён до использования. Оператор `:=` создаёт символ, если имя отсутствует в таблице символов.

## Проверка типов

Проверка выполняется для:

- литералов;
- идентификаторов;
- унарных выражений;
- бинарных выражений;
- присваиваний;
- последовательностей операторов;
- `if`;
- `printf`.

## Условия

Условие `if` должно иметь truthy-compatible тип: `int` или `bool`.

Если `if` содержит `else`, обе ветки анализируются. Несовпадение типов веток считается ошибкой.

## Диагностики

Семантический анализатор накапливает несколько диагностик. Результат анализа возвращается как `TypeCheckResult`.
