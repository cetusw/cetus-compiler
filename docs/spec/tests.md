## Роль тестов

Тесты являются частью семантики CETUS. Целевая модель запрещает запуск программы, если тесты не прошли успешно.

## Property-based testing

Целевая модель должна поддерживать проверку свойств:

```cetus
test "commutative_add" {
    forall(a int, b int) {
        assert(add(a, b) == add(b, a));
    }
}
```

## Изоляция

Каждый тест должен выполняться в изолированном окружении. Изменения состояния одного теста не должны влиять на другой тест.

## Verification phases

Целевая последовательность:

1. синтаксический анализ;
2. семантический анализ;
3. обнаружение тестов;
4. выполнение тестов;
5. проверка покрытия и обязательных инвариантов;
6. запуск программы.
