## Исходный текст

Исходный файл CETUS рассматривается как последовательность символов. Лексический анализатор преобразует её в последовательность токенов.

Лексический анализатор классифицирует входной текст по типам токенов.

## Пробельные символы

Пробельные символы разделяют токены и не попадают в грамматику.

К пробельным символам относятся:

- пробел;
- табуляция;
- перевод строки;
- возврат каретки.

> TODO
>
> Перевод строки не является разделителем операторов. Завершение простого оператора выполняется через `;`.
> 
> Может быть избавимся от этого.

## Комментарии

CETUS поддерживает комментарии:

- однострочный комментарий `//`;
- многострочный комментарий `/* ... */`.

## Идентификаторы

Идентификатор начинается с буквы или `_` и может содержать буквы, цифры и `_`.

Примеры:

```cetus
a
user_id
value123
```

Регистр символов значим.

## Ключевые слова

Текущий лексер распознаёт следующие ключевые слова:

```text
package import func struct var const
if else for range break continue switch case default match return defer
true false nil
printf
```

Слова `int`, `float`, `string`, `bool` зарезервированы для системы типов.

> TODO
>
> После введения деклараций типов встроенные типы должны получить отдельное нормативное описание в лексике и семантике.

## Литералы

Реализованы:

- `INT_LIT`;
- `FLOAT_LIT`;
- `STRING`;
- `TRUE`;
- `FALSE`.

Целочисленный литерал:

```text
123
```

Вещественный литерал:

```text
10.25
```

Логические литералы:

```text
true
false
```

Строки являются частью лексической структуры.

## Операторы и разделители

Операторы и разделители:

```text
( ) { } [ ] . , ;
:=
! != == < <= > >=
+ - * / %
&& ||
```

Зарезервированные операторы:

```text
= ++ -- & | << >> &^ ?. ?? ...
```

> TODO
>
> Для зарезервированных операторов требуется отдельная спецификация синтаксиса и семантики.

## Ошибки лексического анализа

Лексическая ошибка должна содержать строку и описание причины. После первой лексической ошибки дальнейший синтаксический анализ не выполняется.
